# FH_STREAM 固件升级工具使用说明

## 1. 软件简介

FH_STREAM 固件升级工具是一款用于通过串口对嵌入式设备进行固件升级的桌面应用程序。本工具采用 FH_STREAM 通信协议，支持数据分帧传输、CRC校验、自动重传等特性，确保固件升级的可靠性和完整性。
### 主要功能
- 串口通信管理（自动识别可用串口）
- 固件文件加载（支持 .bin 格式）
- 固件数据分帧传输
- 自动重传机制（可配置重传次数和超时时间）
- 实时进度显示
- 双日志窗口（系统日志 + RX数据日志）
- RX数据实时监控（支持字符串/16进制显示）
- 时间戳功能（支持帧合并显示）
- 配置自动保存
## 2. 系统要求
操作系统：Windows 7/10/11
## 3. 安装与运行
运行.exe文件
## 4. 操作步骤
### 5.1 准备工作
使用串口线连接电脑与目标设备

确认设备已上电并处于升级模式

在电脑设备管理器中确认串口号（如 COM3、COM5 等）
### 5.2 软件配置
#### 选择串口
点击串口下拉框选择正确的 COM 口

若无串口显示，请点击 [刷新] 按钮重新扫描

软件会自动记住上次使用的串口号
#### 设置波特率
选择与设备匹配的波特率

常用波特率：9600、19200、38400、57600、115200

默认值：115200
### 5.3 加载固件
点击 [浏览...] 按钮

在文件对话框中选择需要升级的 .bin 固件文件

系统日志区会显示加载成功信息和文件大小
```javascript
[14:30:16] 加载成功: C:\firmware\app.bin (10240 bytes)
```
### 5.4 打开串口
确认串口号和波特率设置正确

点击 [打开串口] 按钮

按钮文字变为 [关闭串口]，表示串口已成功打开

系统日志区显示连接成功信息
```javascript
[14:30:15] 正在打开串口 COM3...
[14:30:16] 串口已打开: COM3 @ 115200
```
### 5.5 开始升级
#### 确认条件
✅ 串口已打开（按钮显示"关闭串口"）

✅ 固件文件已加载（文件路径框不为空）

✅ 开始升级按钮可用（非灰色）
#### 执行升级
点击 [开始升级] 按钮

观察进度条和日志信息

系统日志区会显示每帧的发送和确认信息

RX日志区可实时查看设备返回的数据
#### 升级流程日志示例
```javascript
[14:30:17] 使用参数: 重传次数=5, 超时时间=1000ms
[14:30:17] 固件大小: 10240 bytes, 分为 41 帧
[14:30:17] 发送第 1/41 帧 (ID=0) 尝试 1/5
[14:30:17] 收到ACK, ID=0, VALUE=0x00000000
[14:30:18] 发送第 2/41 帧 (ID=1) 尝试 1/5
[14:30:18] 收到ACK, ID=1, VALUE=0x00000000
[14:30:18] 发送第 3/41 帧 (ID=2) 尝试 1/5
[14:30:18] 收到ACK, ID=2, VALUE=0x00000000
...
[14:30:25] 所有数据帧发送完成，发送结束命令（固件CRC-32）...
[14:30:25] 固件CRC-32: 0x12345678
[14:30:25] 结束命令收到ACK，VALUE=0x00000000，固件校验成功！
[14:30:25] 固件升级成功！共 41 帧，固件CRC=0x12345678
```
#### 完成提示
升级成功后弹出 "完成" 提示框

进度条显示 100%

系统日志区显示成功信息
### 4.6 停止升级
升级过程中可随时点击 [停止] 按钮中断升级

停止后无法恢复，需要重新开始升级

停止后系统日志区会显示"升级已停止"
### 4.7 关闭串口
升级完成后建议点击 [关闭串口] 按钮释放串口资源

关闭串口后按钮文字恢复为 [打开串口]

系统日志区显示"串口已关闭"
## 5. RX日志功能详解
### 5.1 显示模式切换
RX日志支持两种数据显示模式，可通过单选按钮切换：
#### 字符串模式
将接收数据按 UTF-8 解码显示

不可打印字符显示为 \xXX 格式

解码失败时自动切换为十六进制显示

| 示例 | 说明 |
|------|------|
| Hello World! | 正常文本显示 |
| Hello\x0AWorld | 包含换行符的文本 |
| [HEX] 48 65 6C 6C 6F | 解码失败时显示十六进制 |
#### 16进制模式
以十六进制格式显示每个字节

每个字节后都有空格（包括最后一个）

便于复制粘贴和分析协议数据

| 示例 | 说明 |
|------|------|
| 48 65 6C 6C 6F 20 57 6F 72 6C 64 21 | "Hello World!" 的十六进制表示 |
| 55 00 04 00 00 00 00 01 02 03 04 | 协议帧数据 |
#### 切换模式注意事项
切换模式时会自动刷新所有已显示的数据

历史数据会按新模式重新格式化显示

显示模式设置会自动保存，下次启动时恢复
## 5.2 时间戳功能
时间戳功能用于记录每条RX数据的时间信息。
#### 启用时间戳
勾选 [时间戳] 复选框

每条RX数据前会显示毫秒级时间戳

时间戳格式：[HH:MM:SS.zzz]

显示示例：
```javascript
[14:30:25.123] Hello World!
[14:30:25.456] Device response
[14:30:25.789] 55 00 04 00 00 00 00
```
#### 帧合并功能
设置帧间隔阈值（1-9999ms）

连续接收的数据间隔小于阈值时，合并为一帧显示

合并后使用第一条数据的时间戳

适用于将连续发送的多包数据合并显示

合并示例：
```javascript
阈值 100ms，连续收到 3 包数据（间隔 50ms）：
[14:30:25.123] Hello World! How are you? I am fine.
```
#### 关闭时间戳
取消勾选 [时间戳] 复选框

数据连续显示，不换行

不显示时间戳前缀

多条消息会连续追加在同一行
```javascript
Hello World!How are you?I am fine.
```
## 6. 常见问题
### 6.1 串口打开失败
#### 可能原因：
设备未正确连接

串口被其他程序占用

驱动未正确安装

#### 解决方法：
检查设备是否已连接

关闭其他可能占用串口的程序（如串口调试助手）

尝试重新插拔串口线

点击 [刷新] 按钮重新扫描串口

检查设备管理器中的驱动状态
### 6.2 升级超时
#### 可能原因：
串口连接不稳定

超时时间设置过短

波特率不匹配

下位机未正常响应
#### 解决方法：
检查串口连接是否稳定

适当增加超时时间设置（如 2000ms）

确认波特率与设备匹配

检查下位机程序是否正常运行

尝试降低波特率提高稳定性
### 6.3 固件加载失败
#### 可能原因：
文件格式不正确

文件被其他程序占用

文件损坏
#### 解决方法：
确认文件格式为 .bin

关闭可能占用该文件的程序

重新生成或获取固件文件

检查文件是否完整
### 6.4 CRC校验失败
#### 可能原因：
固件文件与设备不匹配

传输过程中数据丢失

串口通信质量差
#### 解决方法：
确认固件文件与设备匹配

检查串口通信质量

尝试降低波特率重新升级

检查下位机固件接收缓冲区大小
### 8.5 下位机无响应
#### 可能原因：
设备未进入升级模式

通信参数不匹配

连接线缆问题
#### 解决方法：
确认设备已进入升级模式

检查波特率、数据位、停止位等参数

检查串口线是否正常

使用串口调试助手测试通信
